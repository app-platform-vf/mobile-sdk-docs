# Release Notes — VDOMiniApp Android 2.2.0-rc10

**Version:** 2.2.0-rc10
**Platform:** Android
**Build:** 20260624130418 (2026-06-24 13:04:18)

## Changelog

### Added

- `MiniAppEvent.SAVE_IMAGE_TO_GALLERY` — JS bridge event: mini app yêu cầu SDK lưu ảnh vào thư viện thiết bị. Payload `SaveImageToGalleryParam(type, data)` với `type` là `"url"` hoặc `"base64"` và `data` là URL string hoặc chuỗi base64 tương ứng.
- `MiniAppEvent.SAVE_FILE` — JS bridge event: mini app yêu cầu SDK tải file về thiết bị. Payload `SaveFileParam(url, fileName)`.
- `UrlPattern(pattern: String, type: String)` — `@Parcelize data class` mới tại `vn.viettelpay.miniappsdk.common.core`. Định nghĩa pattern dùng để intercept navigation trong child mini-app WebView; khi URL (đã decode) chứa `pattern` làm substring, SDK thoát mini app và trả `type` về host.
- `ChildMiniAppConfig.urlPatterns: List<UrlPattern>` — danh sách `UrlPattern` áp dụng cho child mini-app session, default `emptyList()`. Khi non-empty, SDK match decoded URL trước khi fallback về `returnUrl`/`cancelUrl`.
- `StatusBarMode` enum (`DISPLAY`, `HIDDEN`, `TRANSPARENT`) — cấu hình status bar độc lập với `ToolbarMode`.
- `MiniAppThemeConfig.statusBarMode: StatusBarMode?` — field mới, được `MiniAppThemeConfig.default()` set sẵn `StatusBarMode.DISPLAY`. Có thể truyền qua JSON với key `statusBarMode`.
- `MiniAppThemeConfigDto.statusBarMode: String?` — field tương ứng phía DTO, parse vào `MiniAppThemeConfig.statusBarMode`.

### Changed

- Status bar visibility — không còn được suy ra từ `toolbarMode` nữa mà điều khiển bởi `statusBarMode`. `ToolbarMode` giờ chỉ ảnh hưởng tới header bar (`toolbarHeader`, `toolbarActionContainer`).
- `MiniAppFragment.setupStatusBar` — trên API < 30, header transparent giờ dựa vào `statusBarMode == TRANSPARENT` thay vì `toolbarMode`; thêm cập nhật top margin cho `miniAppWebView` khi `toolbarMode` là `HIDDEN`/`TRANSPARENT`.
- WebView callback URL handling — `MiniAppWebViewClient` hợp nhất logic `returnUrl`/`cancelUrl` cũ với matching theo `urlPatterns` mới. `urlPatterns` được ưu tiên match trước (theo thứ tự khai báo); chỉ decode URL khi có callback config để tránh chi phí không cần thiết.
- Insets handling — `MiniAppFragment` và `VdoPermissionConfigFragment` đều thêm `ViewCompat.requestApplyInsets(...)` sau khi cài đặt listener, và mở rộng nhánh xử lý cho API ≥ 30 (R+) thay vì chỉ Vanilla Ice Cream.

### Removed

- `MiniAppFragment` — không còn ẩn/hiện `fakeStatusBar` và gọi `themeController.showStatusBar()/hideStatusBar()` từ nhánh `toolbarMode`. Toàn bộ điều khiển status bar đi qua `statusBarMode` và `updateStatusBarVisibility(config)`.
