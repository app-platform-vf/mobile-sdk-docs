# Release Notes — VDOMiniApp Android 2.2.0-rc7

**Version:** 2.2.0-rc7
**Platform:** Android
**Build:** 20260617092108 (2026-06-17 09:21:08)

## Changelog

### Added

- `MiniAppEvent.INIT_REQUEST` — JS bridge event: mini app yêu cầu SDK gửi lại event `INIT` kèm init data. SDK sẽ phản hồi với payload từ `MiniAppConfig.initRequest` (yêu cầu `MiniAppManifest.exchangeKey = true`).
- `MiniAppDisplay.getMiniAppActivity(): Activity?` — trả về `Activity` đang host mini app (nullable nếu mini app không chạy trong Activity context).
- `HostAppBridge.provideAdId(): String?` (suspend) — host app cung cấp Google Play Services Advertising ID (gps_adid) cho tracking. Trả `null` nếu không khả dụng.

### Changed

- Tracking device identifier — khi `provideAdId()` trả về giá trị non-empty, tracking event gửi field `gps_adid`; ngược lại fallback về `idfv`.
- Keyboard handling — refactor để dùng `OnApplyWindowInsetsListener` với IME insets trên Android R+ (API 30+), giữ `OnGlobalLayoutListener` cho các phiên bản cũ hơn. Hỗ trợ đúng edge-to-edge layout.
- Navigation bar contrast — khi ẩn navigation bar, `window.isNavigationBarContrastEnforced = false`; khi hiện, `= true`.
- `MiniAppFragment` — khi không dùng fake status bar, tự áp dụng `systemBars` insets cho `toolbarActionFrameBackground` và `toolbarHeader`.
- `fragment_mini_app_v2.xml` — bỏ `android:fitsSystemWindows="false"` để hệ thống xử lý insets nhất quán hơn.

### Removed

- `HostAppBridge.provideIdfv(): String?` — thay bằng `provideAdId()` (suspend).
- `HostAppBridge.provideIdfvSuspend(): String?` — thay bằng `provideAdId()` (suspend).
