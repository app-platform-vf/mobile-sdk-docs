# Release Notes - VDOMiniApp Android 2.2.0-rc3

**Version:** 2.2.0-rc3
**Platform:** Android

## Changelog

### Added
- `MiniAppEvent.OPEN_EXTERNAL_LINK` — JS bridge event to open an external URL from a mini app
- `MiniAppEvent.OPEN_MINI_APP` — JS bridge event to open another mini app from within a running mini app
- `MiniAppEvent.MINI_APP_TOKEN` — JS bridge event to retrieve the mini app login token
- `MiniAppEvent.UPDATE_MINI_APP_THEME` — JS bridge event to update the toolbar color dynamically
- `ErrorCode.SDK002` — new SDK error code
- `ChildMiniAppConfig` — Parcelable config for child mini app navigation: `returnUrl`, `cancelUrl`, `closeOnGoBack`
- `MiniAppLocation` — Parcelable data class for location coordinates: `latitude`, `longitude`
- `MiniApp.getChildMiniAppConfig()` — retrieve the child mini app navigation config
- `MiniAppManifest.merchantName` — merchant display name
- `MiniAppManifest.attribute` — UI customization and integration partner data (`Attribute` data class)
- `MiniAppManifest.items` — list of contextual menu items (`MiniAppMenuItem`)
- `MiniAppManifest.versionCode` — version string for the manifest
- `Attribute` data class: `colorCode`, `highlightText`, `integrationPartnerAttributeId`, `integrationPartnerId`, `termAndConditionsText`, `termsLink`
- `MenuItem` data class: `key` (`MiniAppMenuActionKey`), `value`, `iconUrl`
- `Permission.approvalType` (`ApprovalType`) — approval type for the permission
- `Permission.permissionType` — integer category: 1=SDK, 2=device, 3=user data
- `Permission.showTerms` / `Permission.showCheckBoxTerms` — controls terms visibility in the permission bottom sheet
- `InitMiniAppException` — new exception class with `code: String?`, `toCode()`, `isBusinessError()`, `isServerError()`, `isOtherError()`
- `MiniAppConfig.containerId` — `@IdRes Int` container view ID for embedding the mini app (default `-1`)
- `MiniAppConfig.needParams` — controls whether init params are required (default `true`)
- Consumer ProGuard rules (`consumer-rules.pro`) bundled in the AAR for consuming apps
- `androidx.work:work-runtime-ktx:2.8.1` added as a dependency

### Changed
- `compileSdk` bumped from 34 to **35**
- `HostView` constructor: `activity: FragmentActivity` replaced with `context: Context`; `activity` is now a derived nullable property (`context as? FragmentActivity`); `miniAppHeaderConfig` type changed from `MiniAppHeaderConfig` to `MiniAppThemeConfig`
- `MiniAppConfig.headerConfig` renamed to `themeConfig`; type changed from `MiniAppHeaderConfig` to `MiniAppThemeConfig`
- `SdkConfig` simplified — `hostAppApplicationId`, `deviceId`, `imei`, `osVersion`, and `appVersion` removed; these values are now resolved internally by the SDK
- `SdkException` primary constructor now accepts an optional third parameter `code: String?`

### Removed
- `HostAppUser` data class removed from the public API
- `play.services.location` dependency removed

## Requirements

- Gradle 8.0+
- minSdk 24
- compileSdk 35 / targetSdk 34
