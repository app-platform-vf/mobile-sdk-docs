# Release Notes - iOS SDK 2.2.0-rc6

**Version:** 2.2.0-rc6
**Release date:** TBD

## Changelog

### Added
- `MiniAppStatusBarMode` enum (`.display`, `.hidden`, `.transparent`) — controls status bar visibility independently of `toolbarMode`
- `statusBarMode` parameter on `VDOMiniAppThemeConfig.init` (defaults to `.display`)
- `getLocation` method on `VDOHostAppBridge` protocol — host app supplies current device location to the Mini App via `MiniAppLocation?`
- `MiniAppLocation` now conforms to `Equatable` and `Codable`; `latitude` and `longitude` exposed as `public let`
- `VDOMiniAppSdk.exitMiniApp(with:)` — public API to exit a running Mini App by key
- JS bridge event `INIT_REQUEST` (`"INIT_REQUEST"`)

### Changed
- `MiniAppToolbarMode` now controls **only the header bar**; status bar visibility moved to the new `statusBarMode` field. Previously `.hidden` and `.transparent` affected both the status bar and header bar together. Host apps relying on `toolbarMode = .hidden` to also hide the status bar must now set `statusBarMode = .hidden` explicitly.
- Documentation clarified for `MiniAppToolbarMode` cases (`.normal`, `.hidden`, `.transparent`).
