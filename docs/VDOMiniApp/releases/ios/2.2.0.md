# Release Notes - iOS SDK 2.2.0

**Version:** 2.2.0
**Release date:** TBD

## Changelog

### Added
- JS bridge event `MINI_APP_TOKEN` (`miniAppToken`) — retrieve the current mini app login token.
- JS bridge event `UPDATE_MINI_APP_THEME` (`updateMiniAppTheme`) — update the full mini app theme/header at runtime.
- JS bridge event `SAVE_IMAGE_TO_GALLERY` (`saveImageToGallery`) — save an image (from URL or Base64) to the device's photo library.
- JS bridge event `SAVE_FILE` (`saveFile`) — save a file from URL to the device's Documents directory.
- JS bridge event `SET_SWIPE_GESTURE_STATE` (`setSwipeGestureState`) — unified single event to enable/disable swipe gesture.
- JS bridge event `INIT_REQUEST` (`initRequestEvent`) for V1 bridge.
- Public `MiniAppLocation` struct (WGS-84 coordinates) replacing the previous location representation passed through `VDOHostAppBridge.getLocation`.
- Public `VDOMiniAppThemeConfig` model unifying header/toolbar/status bar theming, with enums `MiniAppLeftButton`, `MiniAppToolbarMode`, `MiniAppStatusBarMode`, `MiniAppStatusBarForeground`, and `MiniAppActionButtonThemeType`. `MiniAppThemeConfig` is kept as a backward-compatible typealias.

### Changed
- `MiniApp.getMiniDeepLinkUrlParameters()` return type changed from `[String: String]` to `[String: AnyCodableValue]` to support richer deep-link parameter values.

### Removed
- `VDONavigationBarConfig` — replaced by `VDOMiniAppThemeConfig`.
- JS bridge events `SET_BACKGROUND_STATUS_BAR_COLOR`, `UPDATE_STATUS_BAR_APPEARANCE`, `UPDATE_NAVIGATION_BAR_APPEARANCE`, `SET_NAVIGATION_BAR_COLOR` — consolidated into `UPDATE_MINI_APP_THEME`.
- JS bridge events `SET_SWIPE_GESTURE_LEFT_STATE` and `SET_SWIPE_GESTURE_RIGHT_STATE` — replaced by the single `SET_SWIPE_GESTURE_STATE`.

> Breaking changes — see the [2.1.0 → 2.2.0 migration guide](../../migration/ios/2.1.0-to-2.2.0.md).
