# Migration Guide: Android 2.0.20 → 2.2.0

## Breaking Changes

### 1. `SdkConfig` — fields removed

The following fields have been removed. The SDK now resolves these values internally.

**Before (2.0.20):**
```kotlin
val sdkConfig = SdkConfig(
    hostAppApplicationId = packageName,
    hostAppApplicationName = getString(R.string.app_name),
    isLoadFromAsset = false,
    appVersion = getAppVersion(applicationContext),
    osVersion = getOSVersion(),
    deviceId = Toolbox.getUUID(applicationContext),
    imei = Toolbox.getUUID(applicationContext),
    apiKey = "<your-api-key>",
    variant = BuildVariant.Uat,
    partnerKey = "<your-partner-key>",
    secretId = "<your-secret-id>",
)
```

**After (2.2.0):**
```kotlin
val sdkConfig = SdkConfig(
    hostAppApplicationName = getString(R.string.app_name),
    isLoadFromAsset = false,
    apiKey = "<your-api-key>",
    variant = BuildVariant.Uat,
    partnerKey = "<your-partner-key>",
    secretId = "<your-secret-id>",
)
```

**Removed fields:** `hostAppApplicationId`, `deviceId`, `imei`, `osVersion`, `appVersion`.

---

### 2. `HostAppBridge` — interface overhaul

`HostAppBridge` is now a direct interface to implement. All callback parameters change from `miniAppId: Int` to `miniAppKey: String`. `getUserData` is now async — result must be delivered via `resultHandler`. `intercept` no longer returns `Boolean` and no longer receives a `MiniAppBridgeExecutor` parameter.

**Before (2.0.20):**
```kotlin
val hostAppBridge = HostAppBridgeImpl(hostApp = object : HostAppListener {
    override fun getUserData(miniAppKey: String, dataName: List<String>): Map<String, Any?> {
        return mapOf(...)
    }
    override fun expiredSession(miniAppId: Int, data: Map<String, Any>) {}
    override fun observeLifecycle(miniAppId: Int, miniAppLifecycle: MiniAppLifecycle) {}
    override fun paymentStart(miniAppId: Int, request: String) {}
    override fun paymentEnd(miniAppId: Int, request: String) {}
    override fun paymentRequest(miniAppId: Int, data: Map<String, Any>) {}
    override fun intercept(message: String, executor: MiniAppBridgeExecutor): Boolean = false
})
```

**After (2.2.0):**
```kotlin
class MyHostAppBridge : HostAppBridge {
    override fun getUserData(
        miniAppKey: String,
        dataName: List<String>,
        resultHandler: (Map<String, Any?>) -> Unit
    ) {
        resultHandler(mapOf(...))
    }
    override fun expiredSession(miniAppKey: String, data: Map<String, Any>) {}
    override fun observeLifecycle(miniAppKey: String, miniAppLifecycle: MiniAppLifecycle) {}
    override fun paymentStart(miniAppKey: String, request: String) {}
    override fun paymentEnd(miniAppKey: String, request: String) {}
    override fun paymentRequest(miniAppKey: String, data: Map<String, Any>) {}
    override fun intercept(miniAppKey: String, message: String) {}
    override fun onReturnHome(miniAppKey: String, data: String?) {}
    override fun getLocation(resultHandler: (MiniAppLocation?) -> Unit) { resultHandler(null) }
    override fun tearDown() {}
}

MiniAppSdk.getInstance().hostAppBridge = MyHostAppBridge()
```

**Tóm tắt thay đổi:**

- `HostAppBridgeImpl(hostApp: HostAppListener)` đã bị xóa; implement `HostAppBridge` trực tiếp
- Tất cả `miniAppId: Int` → `miniAppKey: String`
- `getUserData`: nay là void, kết quả trả qua `resultHandler: (Map<String, Any?>) -> Unit`
- `intercept`: signature đổi từ `intercept(message, executor): Boolean` thành `intercept(miniAppKey, message): Unit`
- Methods mới: `onReturnHome`, `getLocation`, `tearDown`

### 3. `HostView` — constructor changed

**Before (2.0.20):**
```kotlin
HostView(
    activity = this,                       // FragmentActivity
    containerId = R.id.container,
    miniAppHeaderConfig = myHeaderConfig   // MiniAppHeaderConfig?
)
```

**After (2.2.0):**
```kotlin
HostView(
    context = this,                        // Context
    containerId = R.id.container,
    miniAppHeaderConfig = myThemeConfig    // MiniAppThemeConfig?
)
```

- Thay `activity` bằng `context`. Property `activity` vẫn tồn tại dưới dạng nullable derived value (`context as? FragmentActivity`).
- `miniAppHeaderConfig` nay nhận `MiniAppThemeConfig` thay vì `MiniAppHeaderConfig`.

---

### 4. `MiniAppConfig.headerConfig` renamed to `themeConfig`

**Before (2.0.20):**
```kotlin
MiniAppConfig(
    initRequest = initRequest,
    headerConfig = myHeaderConfig   // MiniAppHeaderConfig?
)
```

**After (2.2.0):**
```kotlin
MiniAppConfig(
    initRequest = initRequest,
    themeConfig = myThemeConfig     // MiniAppThemeConfig?
)
```

---

### 5. `MiniAppLifecycle` — state names changed

| Before (2.0.20)              | After (2.2.0)                          |
| ---------------------------- | -------------------------------------- |
| `MiniAppLifecycle.Started`   | `MiniAppLifecycle.Initialization`      |
| `MiniAppLifecycle.Resumed`   | `MiniAppLifecycle.RunningInForeground` |
| `MiniAppLifecycle.Paused`    | `MiniAppLifecycle.RunningInBackground` |
| `MiniAppLifecycle.Stopped`   | `MiniAppLifecycle.Unloading`           |
| _(new)_                      | `MiniAppLifecycle.Error(msg: String)`  |

Update your `observeLifecycle` implementation:

```kotlin
override fun observeLifecycle(miniAppKey: String, miniAppLifecycle: MiniAppLifecycle) {
    when (miniAppLifecycle) {
        is MiniAppLifecycle.Initialization -> { }
        is MiniAppLifecycle.RunningInForeground -> { }
        is MiniAppLifecycle.RunningInBackground -> { }
        is MiniAppLifecycle.Error -> { Log.e("MiniApp", miniAppLifecycle.msg) }
        is MiniAppLifecycle.Unloading -> { }
    }
}
```

---

### 6. `openMiniAppFromLinkQR` renamed to `openMiniAppFromDeepLink`

**Before (2.0.20):**
```kotlin
MiniAppSdk.getInstance().openMiniAppFromLinkQR(initInfo, hostView, qrLink, params)
```

**After (2.2.0):**
```kotlin
// Mở trong Fragment:
MiniAppSdk.getInstance().openMiniAppFromDeepLink(
    initInfo = initInfo,
    hostView = hostView,
    qrContent = qrLink
)
// Mở trong Activity mới:
MiniAppSdk.getInstance().openMiniAppActivityFromDeepLink(
    initInfo = initInfo,
    hostView = hostView,
    qrContent = qrLink
)
```

---

### 7. `HostAppUser` removed

The `HostAppUser` data class has been removed from the public API. Remove any references to it.

---

### 8. `SdkException` constructor signature changed

If you subclass or directly construct `SdkException`, the primary constructor now requires `code: String?` as a third parameter:

```kotlin
// Primary constructor (after):
SdkException(message = "error", cause = e, code = null)

// Secondary constructors (message, cause) and (message) are unchanged.
```

---

### 9. `compileSdk` bumped to 35

```kotlin
android {
    compileSdk = 35
}
```

---

## New Features Summary

| Feature | Type | Mô tả |
| ------- | ---- | ----- |
| `MiniAppEvent.OPEN_EXTERNAL_LINK` | JS bridge event | Mở URL bên ngoài |
| `MiniAppEvent.OPEN_MINI_APP` | JS bridge event | Mở mini app khác |
| `MiniAppEvent.MINI_APP_TOKEN` | JS bridge event | Lấy login token |
| `MiniAppEvent.UPDATE_MINI_APP_THEME` | JS bridge event | Cập nhật màu toolbar |
| `ChildMiniAppConfig` | Data class | Cấu hình navigation cho child mini app (`returnUrl`, `cancelUrl`, `closeOnGoBack`) |
| `MiniAppLocation` | Data class | Tọa độ vị trí (`latitude`, `longitude`) |
| `InitMiniAppException` | Exception | Exception có typed error code với `isBusinessError()`, `isServerError()`, `isOtherError()` |
| `MiniAppManifest.attribute` | Field | Dữ liệu UI customization (`Attribute`) |
| `MiniAppManifest.merchantName` | Field | Tên merchant hiển thị |
| `MiniAppManifest.items` | Field | Danh sách menu items (`MiniAppMenuItem`) |
| `MiniAppManifest.versionCode` | Field | Version string của manifest |
| `MiniAppConfig.containerId` | Field | Container view ID cho embedding |
| `MiniAppConfig.needParams` | Field | Toggle init params |
| `ErrorCode.SDK002` | Error code | Mã lỗi SDK mới |
