# Migration Guide: 2.2.0-rc3 → 2.2.0-rc7

Tài liệu các breaking changes khi nâng cấp VDOMiniApp Android từ `2.2.0-rc3` lên `2.2.0-rc7`.

## HostAppBridge — tracking identifier API

`provideIdfv()` và `provideIdfvSuspend()` đã bị xóa và thay bằng một method suspend duy nhất `provideAdId()` trả về Google Play Services Advertising ID.

### Trước (rc3)

```kotlin
class MyHostAppBridge : HostAppBridge {
    override fun provideIdfv(): String? = "<idfv>"
    override suspend fun provideIdfvSuspend(): String? = "<idfv>"
}
```

### Sau (rc7)

```kotlin
class MyHostAppBridge : HostAppBridge {
    override suspend fun provideAdId(): String? {
        // Lấy gps_adid từ Google Play Services Ads Identifier
        return AdvertisingIdClient.getAdvertisingIdInfo(context).id
    }
}
```

### Behavior thay đổi

- Khi `provideAdId()` trả về giá trị non-empty, SDK gửi field `gps_adid` trong tracking event.
- Khi `provideAdId()` trả về `null` hoặc rỗng, SDK fallback về `idfv` (lấy từ payload của mini app).
- Nếu host app không override `provideAdId()`, tracking sẽ dùng `idfv` như cũ.

## Cập nhật dependency

```kotlin
// build.gradle.kts
implementation("vn.viettelpay:miniappsdk:2.2.0-rc7")
```

## Không breaking nhưng nên biết

- `MiniAppDisplay.getMiniAppActivity()` mới — có thể dùng để lấy host `Activity` của mini app đang chạy.
- `MiniAppEvent.INIT_REQUEST` mới — mini app có thể yêu cầu SDK re-emit init data (yêu cầu `MiniAppManifest.exchangeKey = true`).
- Keyboard và edge-to-edge layout được xử lý lại trên Android R+ (API 30+); kiểm tra lại các flow có IME nếu host app custom decor view insets.
