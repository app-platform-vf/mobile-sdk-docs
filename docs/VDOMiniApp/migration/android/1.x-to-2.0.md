# Migration Guide: VDOMiniApp Android 1.x → 2.0

## Breaking Changes

### SdkConfig — fields mới bắt buộc

v2.0 thêm `partnerKey` và `secretId` vào `SdkConfig`. Cả hai được cung cấp khi đăng ký trên hệ thống.

```kotlin
// v1.x
val sdkConfig = SdkConfig(
    apiKey = "...",
    ...
)

// v2.0
val sdkConfig = SdkConfig(
    apiKey = "...",
    partnerKey = "<partner-key>",   // MỚI — bắt buộc
    secretId = "<secret-id>",        // MỚI — bắt buộc
    variant = BuildVariant.Uat,      // MỚI — bắt buộc
    ...
)
```

### HostAppBridge — cách khởi tạo thay đổi

`HostAppBridge` nay được khởi tạo qua `HostAppBridgeImpl(hostApp: HostAppListener)`. Cần implement đầy đủ tất cả methods; các callback payment/lifecycle có thể để trống nếu không dùng.

```kotlin
// v2.0
val hostAppBridge = HostAppBridgeImpl(hostApp = object : HostAppListener {
    override fun getUserData(miniAppKey: String, dataName: List<String>): Map<String, Any?> { ... }
    override fun expiredSession(miniAppId: Int, data: Map<String, Any>) {}
    override fun observeLifecycle(miniAppId: Int, miniAppLifecycle: MiniAppLifecycle) {}
    override fun paymentStart(miniAppId: Int, request: String) {}
    override fun paymentEnd(miniAppId: Int, request: String) {}
    override fun paymentRequest(miniAppId: Int, data: Map<String, Any>) {}
    override fun intercept(message: String, executor: MiniAppBridgeExecutor): Boolean = false
})
```

### openMiniApp — thêm tham số `parameter`

`openMiniApp()` nhận thêm tham số tùy chọn `parameter: Map<String, List<String>>` để truyền query param vào URL của mini app.

## APIs mới

| Method | Mô tả |
| ------ | ----- |
| `openMiniAppFromLinkQR(initInfo, hostView, qrLink, params)` | Mở mini app từ Adjust/QR link |
| `HostAppListener.observeLifecycle` | Theo dõi lifecycle của mini app |
| `HostAppListener.paymentStart/End/Request` | Hooks cho luồng thanh toán |
| `HostAppListener.intercept` | Intercept JS bridge message |
| `MiniAppHeaderConfig` | Cấu hình app bar (title, màu nền, màu chữ, ẩn/hiện) |
