# Migration 2.2.0-rc7 → 2.2.0-rc10

## Update Dependency

```kotlin
implementation("vn.viettelpay:miniappsdk:2.2.0-rc10")
```

## Behavior Changes

### Status bar tách khỏi ToolbarMode

Trước rc10, status bar visibility được suy ra từ `MiniAppThemeConfig.toolbarMode`:

- `ToolbarMode.NORMAL` → status bar hiển thị
- `ToolbarMode.HIDDEN` → status bar ẩn
- `ToolbarMode.TRANSPARENT` → status bar hiển thị, header transparent

Từ rc10, status bar được điều khiển độc lập qua `MiniAppThemeConfig.statusBarMode: StatusBarMode?`. `ToolbarMode` giờ chỉ ảnh hưởng tới header bar.

**Migration:** Nếu trước đây dựa vào việc `ToolbarMode.HIDDEN` cũng ẩn status bar, nay cần set thêm `statusBarMode`:

```kotlin
// Trước
MiniAppThemeConfig(
    toolbarMode = ToolbarMode.HIDDEN
    // status bar tự động ẩn
)

// Sau
MiniAppThemeConfig(
    toolbarMode = ToolbarMode.HIDDEN,
    statusBarMode = StatusBarMode.HIDDEN
)
```

`MiniAppThemeConfig.default()` đã được cập nhật bao gồm `statusBarMode = StatusBarMode.DISPLAY`, nên user dùng default sẽ thấy status bar hiển thị như trước với `ToolbarMode.NORMAL`.

## Additive APIs

Các API sau là additive, không yêu cầu thay đổi code hiện tại:

- `MiniAppEvent.SAVE_IMAGE_TO_GALLERY`, `MiniAppEvent.SAVE_FILE` — JS bridge events mới
- `UrlPattern` và `ChildMiniAppConfig.urlPatterns` — bổ sung cách match callback URL cho child mini-app (cơ chế `returnUrl`/`cancelUrl` cũ vẫn hoạt động)
- `StatusBarMode` enum, `MiniAppThemeConfig.statusBarMode`, `MiniAppThemeConfigDto.statusBarMode`

## No Removed Public APIs

Release này không xóa public API. Chỉ có behavior change của status bar như mô tả ở trên.
