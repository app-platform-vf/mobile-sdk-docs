Release Notes — VDOMiniApp Android 2.2.0

Version: 2.2.0 Platform: Android Build: 20260629164557 (2026-06-29 16:45:57)

Changelog​

Tổng hợp thay đổi kể từ official release 2.0.20 (bao gồm các thay đổi đã ship qua 2.2.0-rc3, 2.2.0-rc7, 2.2.0-rc10).

Added​

  • MiniAppEvent.INIT_REQUEST — JS bridge event: mini app yêu cầu SDK phát lại event INIT với init data.
  • MiniAppEvent.EVENT_TRACKING — JS bridge event tracking.
  • MiniAppEvent.OPEN_EXTERNAL_LINK — mini app yêu cầu mở external link.
  • MiniAppEvent.OPEN_MINI_APP — mở mini app khác từ mini app đang chạy.
  • MiniAppEvent.MINI_APP_TOKEN — lấy login token của mini app.
  • MiniAppEvent.UPDATE_MINI_APP_THEME — cập nhật theme/toolbar runtime.
  • MiniAppEvent.SAVE_IMAGE_TO_GALLERY — JS bridge event yêu cầu SDK lưu ảnh vào thư viện thiết bị. Payload SaveImageToGalleryParam(type, data) với type là "url" hoặc "base64".
  • MiniAppEvent.SAVE_FILE — JS bridge event yêu cầu SDK tải file về thiết bị. Payload SaveFileParam(url, fileName).
  • UrlPattern(pattern: String, type: String) — @Parcelize data class mới tại vn.viettelpay.miniappsdk.common.core. Định nghĩa pattern intercept navigation trong child mini-app WebView; khi URL (đã decode) chứa pattern làm substring, SDK thoát mini app và trả type về host.
  • ChildMiniAppConfig — @Parcelize data class mới chứa returnUrl, cancelUrl, closeOnGoBack, và urlPatterns: List<UrlPattern> (default emptyList()).
  • MiniApp.getChildMiniAppConfig(): ChildMiniAppConfig? — method mới trên interface MiniApp.
  • MiniAppLocation(latitude: Double?, longitude: Double?) — @Parcelize data class mới tại vn.viettelpay.miniappsdk.common.core, dùng cho HostAppBridge.getLocation().
  • MiniAppThemeConfig (vn.viettelpay.miniappsdk.common.eventmodel) — model cấu hình giao diện toolbar mới, thay thế MiniAppHeaderConfig legacy ở MiniAppConfig.themeConfig và HostView.miniAppHeaderConfig. Có thể cập nhật runtime qua MiniAppEvent.UPDATE_MINI_APP_THEME.
  • MiniAppThemeConfigDto — DTO tương ứng cho parsing JSON.
  • StatusBarMode enum (DISPLAY, HIDDEN, TRANSPARENT) — cấu hình status bar độc lập với ToolbarMode.
  • MiniAppThemeConfig.statusBarMode: StatusBarMode? — field mới, default StatusBarMode.DISPLAY. Parse từ JSON với key statusBarMode.
  • MiniAppConfig.containerId: Int (@IdRes) — container view ID cho mini app session, default -1.
  • MiniAppConfig.needParams: Boolean? — flag yêu cầu init params, default true.
  • MiniAppManifest.attribute: Attribute? — UI customization và integration partner info (colorCode, highlightText, integrationPartnerAttributeId, integrationPartnerId, termAndConditionsText, termsLink).
  • MiniAppManifest.items: List<MiniAppMenuItem>? — danh sách menu item của mini app.
  • MiniAppManifest.previousMiniAppKey: String? và versionCode: String?.
  • Permission.approvalType: ApprovalType, permissionType: Int?, showTerms: Int?, showCheckBoxTerms: Int? — fields mới phục vụ permission UI.
  • Permission.defaultName — tên hiển thị mặc định theo PermissionCode.
  • MenuItem(key: MiniAppMenuActionKey?, value: String?, iconUrl: String?) — @Parcelize data class mới.
  • Attribute — @Parcelize data class mới cho UI và integration partner metadata.
  • BaseDialogFragment — base dialog fragment theo MVI architecture (intent/state/diff/event/viewmodel).
  • BaseFragment — thêm các hook updateToolbarTheme, setupToolbarBackground, setupToolbarTitle, setupToolbarNavIcon, setupToolbarActionTheme, setupStatusBar, setupBottomNavigationBar, updateStatusBarVisibility cho subclass override.
  • ErrorCode.SDK002 — error code mới.
  • InitMiniAppException(message, code) — exception chuyên biệt cho lỗi init mini app, có toCode(), isBusinessError(), isServerError(), isOtherError().
  • SdkException(message, cause, code) — secondary constructor thêm code: String?.

Changed​

  • HostView constructor — activity: FragmentActivity được thay bằng context: Context. activity trở thành derived property nullable (context as? FragmentActivity). containerId có default -1. miniAppHeaderConfig đổi type từ MiniAppHeaderConfig sang MiniAppThemeConfig?.
  • MiniAppConfig.headerConfig: MiniAppHeaderConfig? được rename thành themeConfig: MiniAppThemeConfig?.
  • Status bar visibility — không còn được suy ra từ toolbarMode nữa mà điều khiển bởi statusBarMode. ToolbarMode giờ chỉ ảnh hưởng tới header bar (toolbarHeader, toolbarActionContainer).
  • MiniAppFragment.setupStatusBar — trên API < 30, header transparent giờ dựa vào statusBarMode == TRANSPARENT thay vì toolbarMode; thêm cập nhật top margin cho miniAppWebView khi toolbarMode là HIDDEN/TRANSPARENT.
  • WebView callback URL handling — MiniAppWebViewClient hợp nhất logic returnUrl/cancelUrl cũ với matching theo urlPatterns mới. urlPatterns được ưu tiên match trước (theo thứ tự khai báo); chỉ decode URL khi có callback config.
  • Insets handling — MiniAppFragment và VdoPermissionConfigFragment thêm ViewCompat.requestApplyInsets(...) sau khi cài đặt listener, mở rộng nhánh xử lý cho API ≥ 30 (R+).
  • BaseFragment.render / handleSingleEvent — bỏ TODO() mặc định, cho phép subclass không override mà không bị crash.
  • compileSdk / targetSdk nâng từ 34 lên 35.
  • 16 KB page size support — CMakeLists.txt thêm -Wl,-z,max-page-size=16384; defaultConfig.externalNativeBuild.cmake.arguments thêm -DANDROID_SUPPORT_FLEXIBLE_PAGE_SIZES=ON.
  • Build pipeline — BuildConfig được bật, expose MINI_APP_SDK_VERSION qua buildConfigField. Thêm consumerProguardFiles("consumer-rules.pro"). Artifact upload chọn *-release.aar hoặc *-debug.aar theo property upload.

Removed​

  • HostAppUser(userId: String) — @Parcelize data class này bị xóa khỏi vn.viettelpay.miniappsdk.common.core.
  • SdkConfig — bỏ các field hostAppApplicationId, deviceId, imei, osVersion, appVersion. Các thông tin device/app được SDK tự thu thập nội bộ.
  • MiniAppFragment (file runtime/MiniAppFragment.kt) và CustomMiniAppFragment legacy bị xóa; thay thế bằng runtime/display/main/MiniAppFragment.kt.
  • MiniAppFragment — không còn ẩn/hiện fakeStatusBar và gọi themeController.showStatusBar()/hideStatusBar() từ nhánh toolbarMode. Toàn bộ điều khiển status bar đi qua statusBarMode và updateStatusBarVisibility(config).
  • Dependency com.google.android.gms:play-services-location bị gỡ; vị trí được host app cung cấp qua HostAppBridge.getLocation().

Migration​

Xem 2.0.20 → 2.2.0 để biết các breaking change và hướng dẫn migrate.
