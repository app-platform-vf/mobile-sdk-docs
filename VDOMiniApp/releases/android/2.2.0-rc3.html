Release Notes — VDOMiniApp Android 2.2.0-rc3

Version: 2.2.0-rc3 Platform: Android

Changelog​

Added​

  • MiniAppEvent.OPEN_EXTERNAL_LINK — JS bridge event: mini app yêu cầu mở URL bên ngoài
  • MiniAppEvent.OPEN_MINI_APP — JS bridge event: mini app yêu cầu mở một mini app khác
  • MiniAppEvent.MINI_APP_TOKEN — JS bridge event: mini app yêu cầu lấy login token
  • MiniAppEvent.UPDATE_MINI_APP_THEME — JS bridge event: mini app yêu cầu cập nhật màu toolbar
  • ErrorCode.SDK002 — mã lỗi SDK mới
  • ChildMiniAppConfig — Parcelable config cho child mini app navigation: returnUrl, cancelUrl, closeOnGoBack
  • MiniAppLocation — Parcelable data class cho tọa độ vị trí: latitude, longitude
  • MiniAppManifest.merchantName — tên merchant hiển thị
  • MiniAppManifest.attribute — dữ liệu UI customization và integration partner (Attribute data class)
  • MiniAppManifest.items — danh sách contextual menu items (MiniAppMenuItem)
  • MiniAppManifest.versionCode — version string của manifest
  • Attribute data class: colorCode, highlightText, integrationPartnerAttributeId, integrationPartnerId, termAndConditionsText, termsLink
  • Permission.approvalType (ApprovalType) — loại approval của permission
  • Permission.permissionType — phân loại permission: 1=SDK, 2=device, 3=user data
  • Permission.showTerms / Permission.showCheckBoxTerms — kiểm soát hiển thị terms trong permission bottom sheet
  • InitMiniAppException — exception class mới với code: String?, toCode(), isBusinessError(), isServerError(), isOtherError()
  • MiniAppConfig.containerId — @IdRes Int container view ID để embed mini app (mặc định -1)
  • MiniAppConfig.needParams — kiểm soát việc yêu cầu init params (mặc định true)
  • HostAppBridge.onReturnHome(miniAppKey, data) — callback khi user nhấn về trang chủ
  • HostAppBridge.getLocation(resultHandler) — cung cấp vị trí thiết bị cho mini app
  • HostAppBridge.tearDown() — dọn dẹp tài nguyên khi SDK bị hủy
  • Consumer ProGuard rules (consumer-rules.pro) bundled trong AAR
  • androidx.work:work-runtime-ktx:2.8.1 thêm vào dependencies

Changed​

  • compileSdk tăng từ 34 lên 35
  • HostAppBridge — interface overhaul:
    • Tất cả miniAppId: Int → miniAppKey: String
    • getUserData nay là void với callback resultHandler: (Map<String, Any?>) -> Unit
    • intercept đổi từ intercept(message, executor): Boolean thành intercept(miniAppKey, message): Unit
  • HostView constructor: activity: FragmentActivity thay bằng context: Context; miniAppHeaderConfig đổi kiểu từ MiniAppHeaderConfig sang MiniAppThemeConfig
  • MiniAppConfig.headerConfig đổi tên thành themeConfig; kiểu đổi từ MiniAppHeaderConfig sang MiniAppThemeConfig
  • MiniAppLifecycle — tất cả state class được đổi tên:
    • Started → Initialization
    • Resumed → RunningInForeground
    • Paused → RunningInBackground
    • Stopped → Unloading
    • Error(msg: String) — state mới
  • openMiniAppFromLinkQR() → openMiniAppFromDeepLink() (tham số qrLink → qrContent)
  • Thêm openMiniAppActivityFromDeepLink() để mở mini app từ deeplink trong Activity mới
  • SdkConfig đơn giản hóa — hostAppApplicationId, deviceId, imei, osVersion, appVersion bị xóa; SDK tự resolve các giá trị này
  • SdkException primary constructor nhận thêm tham số thứ ba code: String?

Removed​

  • HostAppUser data class bị xóa khỏi public API
  • play.services.location dependency bị xóa

Requirements​

  • Gradle 8.0+
  • minSdk 24
  • compileSdk 35 / targetSdk 34

Distribution​

implementation("vn.viettelpay:miniappsdk:2.2.0-rc3")
